# SunEnergyXT 零馈电蓝图运行工况说明书

本文说明当前版本 Home Assistant 蓝图在不同现场工况下的运行逻辑、控制目标和注意事项。它适合安装人员、测试人员和需要理解蓝图行为的用户阅读。

蓝图只控制用户选择的 SunEnergyXT 500 / 500 Pro 一体机。外部电表只作为反馈数据来源，不会被蓝图控制。外部微逆、其他一体机、负载模拟设备等都不应配置到蓝图的一体机设备中。

## 1. 蓝图控制目标

蓝图的目标是根据外部电表实时功率，动态调整 SunEnergyXT 一体机的两个设置值，让整屋对电网的功率尽量接近目标值。

默认目标是：

```text
外部电表功率接近 0 W
```

也就是尽量不向电网馈电，也尽量不从电网购电。高级设置中的 `Target grid power` 可以改变目标值，但普通用户建议保持 `0 W`。

蓝图主要写入这两个实体：

| 一体机实体 | 蓝图用途 |
| --- | --- |
| `System Grid Port Power Setpoint` | 调整并网口功率目标，用于增减对电网侧的输出或吸收 |
| `System Max Inverter Power Setpoint` | 调整逆变器总输出上限，用于限制或释放负载口和并网口可用功率 |

蓝图会读取这些实体：

| 实体 | 用途 |
| --- | --- |
| 外部电表实时功率 | 判断整屋当前是购电还是馈电 |
| `System Load Port Power` | 判断负载口是有负载还是有外部回灌 |
| `System Grid Port Power` | 判断一体机并网口实际功率 |
| `PV Total Input Power` | 判断当前 PV 输入 |
| `System Battery Level` | 判断 SOC 上限、下限和满电状态 |
| `System Min Discharge SOC` | 蓝图同步用户设置的放电下限 |
| `System Max Charge SOC` | 蓝图同步用户设置的充电上限 |

## 2. 功率方向定义

蓝图内部统一使用下面的方向。电表插件的原始方向可以不同，蓝图会按电表预设和符号设置转换。

| 数据 | 正数含义 | 负数含义 |
| --- | --- | --- |
| 外部电表功率 | 家庭向电网馈电 | 家庭从电网购电 |
| 负载口功率 | 负载口正在消耗功率 | 负载口有微逆或外部设备回灌 |
| 并网口功率 | 一体机向并网侧输出 | 并网侧向一体机输入或被吸收 |

如果现场观察到“越调越偏”，优先检查外部电表符号方向。

## 3. 用户主要配置项

普通用户只需要配置以下项目：

| 配置项 | 说明 |
| --- | --- |
| `External meter device` | 选择外部电表设备 |
| `SunEnergyXT device` | 选择被蓝图控制的一体机 |
| `System Min Discharge SOC` | 电池放电下限 |
| `System Max Charge SOC` | 电池充电上限 |
| `Full-battery behavior` | 满电后的运行策略 |
| `Maximum on-grid output power` | 最大允许并网输出功率 |

建议最大并网输出功率：

| 型号 | 建议最大值 |
| --- | --- |
| SunEnergyXT 500 | `800 W` |
| SunEnergyXT 500 Pro | `2400 W` |

如当地法规或现场安装要求更低，应填写更低值。

## 4. 每轮控制过程

蓝图默认每秒触发一次，但不是每秒都写设备。每一轮大致分为以下步骤：

1. 读取外部电表、一体机功率、PV、SOC 和上下限实体。
2. 判断数据是否有效、是否超时。
3. 把电表功率转换成蓝图内部方向。
4. 根据用户设置同步设备 SOC 上下限。
5. 判断当前是否满电保持或低 SOC 保持。
6. 根据电表误差、负载口功率、PV、SOC 状态计算目标输出。
7. 对目标值应用最大并网输出、最小并网功率、AC 耦合回充上限等限制。
8. 判断是否达到写入间隔和写入分辨率。
9. 必要时写入 `System Grid Port Power Setpoint` 和 `System Max Inverter Power Setpoint`。

为了减少来回振荡，蓝图会根据误差大小选择快、中、慢三种写入节奏。大误差时调节更快，小误差时调节更慢。

## 5. SOC 状态和回差

蓝图使用用户设置的一体机 SOC 上下限，并将设置同步到设备自带的上下限实体。

内部固定使用 `5%` 回差：

| 状态 | 进入条件 | 退出条件 |
| --- | --- | --- |
| 满电保持 | SOC 达到 `System Max Charge SOC` | 配置 Helper 时，SOC 低于 `上限 - 5%` |
| 低 SOC 保持 | SOC 达到 `System Min Discharge SOC` | 配置 Helper 时，SOC 高于 `下限 + 5%` |

如果没有配置可选 Helper，蓝图会直接根据当前 SOC 判断状态，不跨 Home Assistant 重启保存中间状态。

## 6. 普通零馈电工况

条件：

- SOC 未达到上限。
- SOC 未达到下限。
- 外部电表、一体机和 PV 数据正常。

这是最常见的运行状态。

| 现场现象 | 蓝图动作 |
| --- | --- |
| 外部电表显示馈电 | 降低输出，必要时让设备吸收多余功率 |
| 外部电表显示购电 | 提高输出，补足家庭负载 |
| PV 输入变化 | 继续以外部电表接近 `0 W` 为目标 |
| 负载增加 | 提高逆变输出或并网口输出 |
| 负载减少 | 降低逆变输出，避免外送 |

在该工况下，蓝图优先追求零馈电。PV 是否被完全利用，取决于当前负载、SOC、设备吸收能力和功率限制。

## 7. 满电工况

SOC 达到用户设置的上限后，蓝图进入满电保持。满电后行为由 `Full-battery behavior` 决定。

### 7.1 Follow load after full

这是更保守的满电策略，适合优先严格零馈电的场景。

运行特点：

- 满电后仍以外部电表接近 `0 W` 为目标。
- 当 PV 大于现场负载时，蓝图会降低可用输出，设备可能限制 PV。
- 如果现场负载增加，蓝图会释放输出补负载。
- 如果外部电表仍显示馈电，蓝图会继续压低输出。

适合：

- 当地不允许并网馈电。
- 用户更重视电表不外送。
- 可以接受满电后 PV 被限制。

### 7.2 Follow PV after full

这是更积极利用 PV 的满电策略。

运行特点：

- 满电后尽量让 PV 通过一体机供应负载口。
- 负载口有负载时，PV 会优先供应负载口。
- PV 大于负载时，剩余 PV 可能通过并网侧外送。
- 蓝图会限制主动设置的并网输出目标，但满电 bypass 的实际硬件路径可能不完全等同于设置值。

适合：

- 用户希望满电后尽量不浪费 PV。
- 现场允许一定并网输出。
- 最大并网输出限制不是唯一安全边界，现场还有其他保护或法规允许。

重要说明：

当前实测中，在满电 `Follow PV after full` 工况下，如果 PV 大于负载，即使蓝图没有主动提高并网口设定，实际电表侧仍可能看到剩余 PV 外送。这更像是设备满电 bypass 路径的硬件行为。如果现场必须严格零馈电，建议使用 `Follow load after full`。

## 8. 低 SOC 工况

SOC 达到用户设置的下限后，蓝图进入低 SOC 保持。

运行特点：

- 蓝图避免继续放电。
- 如果有 PV，输出目标会尽量限制在可用 PV 范围内。
- 如果家庭负载大于 PV，外部电表可能显示购电。
- 蓝图不会为了追求零馈电而突破用户设置的放电下限。

适合观察的实体：

| 实体 | 期望现象 |
| --- | --- |
| `System Battery Level` | 接近或低于下限 |
| `System Max Inverter Power Setpoint` | 不会无限提高 |
| 外部电表功率 | 可能保留一定购电 |

## 9. 外部电表馈电工况

外部电表显示馈电，表示整屋有多余功率正在流向电网。

常见来源：

- PV 大于负载。
- 负载突然降低。
- 负载口有微逆或其他设备回灌。
- 其他外部发电设备在同一回路上输出。

蓝图动作：

| 电池状态 | 动作 |
| --- | --- |
| 未满电 | 尝试吸收多余功率或降低输出 |
| 满电且 Follow load | 压低输出，优先减少外送 |
| 满电且 Follow PV | 优先保持 PV 利用，可能允许剩余 PV 外送 |
| 低 SOC | 不会主动通过电池放电抵消馈电，主要依赖 PV 和限制逻辑 |

如果电表长期保持较大馈电，应检查：

- 电表符号方向是否正确。
- 一体机是否已经满电。
- `Full-battery behavior` 是否选了 `Follow PV after full`。
- `AC-coupled maximum charge power` 是否太低。
- 设备是否因硬件限制无法吸收更多功率。

## 10. 外部电表购电工况

外部电表显示购电，表示整屋负载没有被一体机覆盖。

常见来源：

- 家庭负载突然增加。
- PV 降低。
- 一体机达到最大并网输出功率。
- 电池达到低 SOC 下限。
- 设备实体或写入超时，导致本轮没有成功调节。

蓝图动作：

| 状态 | 动作 |
| --- | --- |
| 电池未到下限 | 提高输出，补足负载 |
| 达到最大并网输出功率 | 不再继续提高并网输出 |
| 达到低 SOC | 避免继续放电，允许购电存在 |
| PV 有输入 | 尽量利用 PV，但不突破 SOC 和输出限制 |

如果购电长期无法收敛，应检查最大并网输出功率是否设置过低。

## 11. 负载口有负载

负载口功率为正，表示一体机负载口正在向负载供电。

蓝图会把负载口功率纳入总输出计算：

- 负载口负载较小时，蓝图可能只需要较低的最大逆变输出。
- 负载口负载增加时，蓝图会提高最大逆变输出。
- 如果外部电表同时购电，蓝图会进一步提高并网口或逆变输出。
- 如果满电选择 `Follow PV after full`，PV 会优先供负载口。

观察点：

| 实体 | 正常现象 |
| --- | --- |
| `System Load Port Power` | 接近负载口实际负载 |
| `System Max Inverter Power Setpoint` | 随负载增加而提高 |
| 外部电表功率 | 最终接近目标值 |

## 12. 负载口有外部回灌

负载口功率为负，表示负载口侧有微逆或其他设备向一体机回灌。

蓝图会把这部分功率视为现场额外输入，并尝试避免它外送到电网。

运行特点：

- 电池未满时，蓝图会尝试吸收或降低其他输出。
- 如果回灌较大，可能受设备硬件能力限制，无法完全吸收。
- `AC-coupled maximum charge power` 会限制部分 AC 侧回充能力。
- 当前实测中，负载口回灌路径和并网口回充限制不完全等同，不能简单理解为所有回灌都被该上限硬限制。

建议：

- 如果现场有负载口微逆，先使用较保守的 `Follow load after full`。
- 如果希望允许满电后 PV 或微逆继续利用，需要确认当地并网限制。
- 测试时重点观察外部电表是否长期馈电。

## 13. PV 输入工况

PV 输入会影响蓝图的输出计算，但蓝图不会直接控制 PV。

未满电时：

- 蓝图优先让外部电表接近 `0 W`。
- 如果 PV 多于现场负载，设备可能吸收或限制输出。
- 如果 PV 少于现场负载，蓝图可能通过电池补足。

满电 Follow load 时：

- 蓝图仍优先零馈电。
- PV 可能被限制。

满电 Follow PV 时：

- 蓝图尽量让 PV 经过设备供应负载口。
- 负载用不完的 PV 可能外送。

低 SOC 时：

- 蓝图会尽量使用 PV。
- 不会为了零馈电而强制从电池继续放电。

## 14. 最大并网输出功率

`Maximum on-grid output power` 限制蓝图主动计算出的并网侧输出目标。

常见配置：

| 设备 | 推荐值 |
| --- | --- |
| SunEnergyXT 500 | `800 W` |
| SunEnergyXT 500 Pro | `2400 W` |

如果填写更低值：

- 蓝图不会主动把并网口输出目标调到该值以上。
- 当家庭负载大于该限制时，可能保留购电。
- 满电 `Follow PV after full` 的 bypass 工况下，实际外送可能受硬件路径影响，不能只依赖该项作为法规限制。

## 15. AC 耦合最大充电功率

`AC-coupled maximum charge power` 用于限制蓝图允许设备通过 AC 侧吸收外部多余功率的能力。

常见影响：

- 设置较高时，蓝图更容易吸收外部馈电。
- 设置较低时，蓝图会减少负向吸收能力。
- 如果外部回灌来自负载口，实际限制还取决于设备硬件路径。

如果现场不希望设备大量吸收外部 AC 侧功率，可以适当降低该值。

## 16. 电表类型和三相数据

蓝图支持总功率、三相求和、import/export 功率对等多种电表形态。

常见情况：

| 电表形态 | 蓝图行为 |
| --- | --- |
| 单个总功率实体 | 直接使用该实体作为外部电表功率 |
| L1/L2/L3 三相实时功率 | 计算三相求和 |
| import/export 功率对 | 计算 `export - import` |
| 三相 import/export | 分别求和后计算净功率 |

Shelly 三相求和模式下，只要任一相功率实体在 `Maximum meter age` 内更新，蓝图就会继续运行。没有变化的相位会沿用当前 state。这是为了兼容部分电表在某一相长期为 `0 W` 时不刷新该相实体的情况。

如果电表设备页只有总功率实体，自动绑定通常会选择总功率。若要使用三相求和，需要在高级设置里手动指定 L1/L2/L3 实体。

## 17. 数据异常和超时保护

蓝图在以下情况下会停止本轮写入：

- 外部电表实体不存在、无效或为 `unknown` / `unavailable`。
- 一体机关键实体无效。
- 电表、负载口、并网口、PV 或 SOC 数据超过 `Maximum meter age` 没有更新。
- 设备上下限实体不可用。

停止本轮写入不等于关闭自动化。下一轮数据恢复后，蓝图会继续计算。

如果看到自动化没有动作，优先检查：

| 检查项 | 说明 |
| --- | --- |
| 自动化是否启用 | HA 自动化开关是否打开 |
| 实体是否可用 | 是否存在 `unknown` / `unavailable` |
| 数据是否刷新 | `last_updated` 是否长期不变 |
| 电表预设是否正确 | 是否选错总功率、三相或 import/export |
| HA 日志 | 是否有插件写入超时 |

## 18. 写入节奏和防振荡

蓝图不会每秒无条件写入设备。它会根据误差大小、写入间隔和分辨率判断是否需要写入。

| 误差状态 | 行为 |
| --- | --- |
| 大误差 | 更快写入，单次步长更大 |
| 中等误差 | 中速修正 |
| 小误差 | 慢速微调，减少来回波动 |

调节后蓝图会等待并网口和电表数据更新，再继续下一次有效修正。这可以减少“刚下发，反馈还没变，又下发下一次”的振荡。

如果仍出现频繁波动，可以考虑：

- 增大 `Small-error threshold`。
- 增大 `Slow write interval` 或 `Medium write interval`。
- 降低 `Correction gain`。
- 检查外部电表和一体机功率上报延迟。

## 19. 当前版本已验证工况

当前版本已在真实 Home Assistant 和设备上验证过以下工况：

| 工况 | 结果 |
| --- | --- |
| 未满电、PV 存在、外部电表自动绑定 | 可收敛到接近零 |
| 外部设备向同回路馈电 | 可收敛到接近零 |
| 外部设备作为负载 | 可收敛到接近零 |
| 显式三相求和，只有一相实时更新 | 可继续运行并按三相 state 求和 |
| 满电 Follow load | 优先零馈电，可能限制 PV |
| 满电 Follow PV | 优先利用 PV，剩余 PV 可能外送 |
| 小并网输出限制 | 可限制蓝图主动并网输出目标，但不能保证限制满电 bypass 实际外送 |

尚未完整真实验证的工况：

| 工况 | 原因 |
| --- | --- |
| 真实低 SOC 到下限 | 当前测试设备 SOC 较高，无法安全制造低 SOC |
| 长时间 7x24 运行 | 需要测试平台持续采集自动化 trace、HA 日志和设备状态 |

## 20. 常见现象解释

| 现象 | 可能原因 |
| --- | --- |
| 满电后 PV 被压低 | 使用了 `Follow load after full`，蓝图优先零馈电 |
| 满电后仍有馈电 | 使用了 `Follow PV after full`，或设备 bypass 路径外送剩余 PV |
| 电表一直购电 | 输出上限太低、低 SOC 保持、PV 不足或写入失败 |
| 电表一直馈电 | 电表方向反、设备满电、回充能力不足或满电 Follow PV |
| 外部微逆回灌吸收不完全 | AC 侧限制、负载口硬件路径或电池满电 |
| 自动化偶尔不写入 | 数据未刷新、未到写入间隔或 HA 插件写入超时 |

## 21. 推荐配置

| 用户目标 | 推荐配置 |
| --- | --- |
| 尽量严格零馈电 | `Follow load after full` |
| 满电后尽量多用 PV | `Follow PV after full` |
| 当地限制并网输出 | 降低 `Maximum on-grid output power`，并优先使用 `Follow load after full` |
| 有负载口微逆 | 先用 `Follow load after full`，观察电表是否外送 |
| 电表数据慢 | 适当增大 `Maximum meter age` |
| 调节太慢 | 缩短写入间隔或提高单次步长 |
| 调节来回波动 | 增大误差阈值、降低增益或延长写入间隔 |

## 22. 重要边界

- 蓝图不能控制外部电表，也不能控制未选择的一体机。
- 蓝图不能直接改变硬件内部 bypass 逻辑。
- 蓝图主动设置值受 `Maximum on-grid output power` 限制，但实际功率还会受设备硬件路径影响。
- 如果设备插件写入超时，本轮设置可能不会成功，需要依赖下一轮重试或 HA 日志排查。
- 如果现场法规要求绝对不外送，不建议使用满电 `Follow PV after full` 作为唯一保护。
