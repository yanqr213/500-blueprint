# SunEnergyXT 零馈电蓝图需求文档

本文档基于当前版本蓝图的产品目标、配置界面和控制行为反向整理，用于明确蓝图应满足的产品需求、功能边界和验收标准。

当前已做过的真实 Home Assistant 验证只作为后续测试方案建议，不作为需求来源本身。相关内容单独放在文档末尾的“建议测试方案”中。

文档不包含任何现场设备序列号、设备 ID、账号、地址或私有网络信息。

## 1. 背景

SunEnergyXT 500 / 500 Pro 一体机接入 Home Assistant 后，可以通过实体控制并网口功率设定和最大逆变输出设定。用户希望使用外部电表作为整屋功率反馈，实现接近零馈电的自动控制。

现有 Home Assistant 插件暴露的实体足以支持本地闭环控制，但不同电表插件、不同语言环境、不同电表功率方向和不同现场接线方式会导致实体名称、功率符号和更新时间不一致。因此蓝图需要尽量降低用户配置成本，同时保留高级覆盖能力。

## 2. 目标

蓝图目标是：

- 通过外部电表反馈，让整屋电表功率尽量接近用户设置的目标值，默认 `0 W`。
- 只要求普通用户选择电表设备、一体机设备、SOC 上下限、满电策略、最大并网输出功率和 AC 耦合最大充电功率。
- 自动绑定一体机关键实体和常见电表实体。
- 支持 SunEnergyXT 500 / 500 Pro 的不同并网输出上限。
- 支持满电、低 SOC、PV 输入、负载口负载、负载口回灌等主要工况。
- 在数据无效、实体不可用或数据超时时停止本轮写入，避免错误控制。
- 为后续自动化测试和 7x24 测试平台提供清晰的需求依据。

## 3. 非目标

蓝图不负责：

- 控制外部电表。
- 控制未被用户选中的其他一体机或外部设备。
- 修改一体机运行模式、放电模式或固件策略。
- 绕过设备硬件限制。
- 保证满电 bypass 硬件路径一定受蓝图主动设定值完全限制。
- 在外部设备、插件或网络异常时保证每次写入都成功。

## 4. 用户角色

| 角色 | 需求 |
| --- | --- |
| 普通用户 | 通过少量配置实现零馈电，避免理解复杂实体 |
| 安装人员 | 能快速判断电表方向、SOC 设置和最大并网输出限制是否正确 |
| 测试人员 | 能按工况验证蓝图控制逻辑、异常保护和边界行为 |
| 售后支持 | 能根据现象定位配置错误、设备限制或插件写入异常 |

## 5. 术语

| 术语 | 含义 |
| --- | --- |
| 外部电表 | Home Assistant 中接入的整屋实时功率电表 |
| 一体机 | 被蓝图控制的 SunEnergyXT 500 / 500 Pro 设备 |
| 并网口功率设定 | `System Grid Port Power Setpoint` |
| 最大逆变输出设定 | `System Max Inverter Power Setpoint` |
| 负载口功率 | `System Load Port Power` |
| 并网口实际功率 | `System Grid Port Power` |
| PV 功率 | `PV Total Input Power` |
| SOC | `System Battery Level` |
| 满电保持 | SOC 达到充电上限后的控制状态 |
| 低 SOC 保持 | SOC 达到放电下限后的控制状态 |

## 6. 功率方向要求

蓝图内部必须统一功率方向：

| 数据 | 正数 | 负数 |
| --- | --- | --- |
| 外部电表功率 | 向电网馈电 | 从电网购电 |
| 负载口功率 | 负载消耗 | 负载口回灌 |
| 并网口功率 | 一体机向并网侧输出 | 并网侧向一体机输入或被吸收 |

需求：

- REQ-PWR-01：蓝图必须支持电表原始方向与内部方向不同的情况。
- REQ-PWR-02：蓝图必须提供自动方向判断和手动方向覆盖。
- REQ-PWR-03：蓝图必须支持功率单位倍率转换，例如 `kW` 转 `W`。

## 7. 用户配置需求

普通配置界面只应暴露必要项：

| 编号 | 需求 |
| --- | --- |
| REQ-UI-01 | 用户必须先选择电表类型或功率计算公式 |
| REQ-UI-02 | 用户必须选择外部电表设备 |
| REQ-UI-03 | 用户必须选择 SunEnergyXT 一体机设备 |
| REQ-UI-04 | 用户必须设置 `System Min Discharge SOC` |
| REQ-UI-05 | 用户必须设置 `System Max Charge SOC` |
| REQ-UI-06 | 用户必须选择满电后的策略 |
| REQ-UI-07 | 用户必须设置最大并网输出功率 |
| REQ-UI-08 | 用户必须设置或确认 AC 耦合最大充电功率 |
| REQ-UI-09 | 高级实体覆盖、符号方向、倍率、写入节奏等设置应默认折叠 |
| REQ-UI-10 | Shelly Pro 3EM、EcoTracker、BitShake / Tasmota 等设备型电表通常只要求用户选择电表类型和电表设备 |
| REQ-UI-11 | 普通 Shelly 3EM 应允许用户选择 L1 + L2 + L3 三相求和类型，并填写三相功率实体 |
| REQ-UI-12 | 自定义电表应允许用户选择功率计算公式，并填写该公式所需实体 |

最大并网输出功率说明：

| 设备型号 | 推荐最大值 |
| --- | --- |
| SunEnergyXT 500 | `800 W` |
| SunEnergyXT 500 Pro | `2400 W` |

如果当地法规或现场要求更低，用户应填写更低值。

## 8. 自动绑定需求

一体机实体自动绑定：

| 编号 | 需求 |
| --- | --- |
| REQ-BIND-01 | 蓝图应从所选一体机设备自动寻找并网口功率设定实体 |
| REQ-BIND-02 | 蓝图应自动寻找最大逆变输出设定实体 |
| REQ-BIND-03 | 蓝图应自动寻找负载口功率、并网口实际功率、PV、SOC 实体 |
| REQ-BIND-04 | 蓝图应自动寻找设备自带 SOC 上下限实体 |
| REQ-BIND-05 | 自动绑定失败或不准确时，用户可在高级设置中手动覆盖 |

外部电表自动绑定：

| 编号 | 需求 |
| --- | --- |
| REQ-METER-01 | 支持常见单个总功率实体 |
| REQ-METER-02 | 支持 Shelly 类电表总功率 |
| REQ-METER-03 | 支持 L1/L2/L3 三相实时功率求和 |
| REQ-METER-04 | 支持总 import/export 功率对 |
| REQ-METER-05 | 支持三相 import/export 功率对 |
| REQ-METER-06 | 支持 BitShake / Tasmota 当前功率实体 |
| REQ-METER-07 | 支持 EcoTracker 当前功率实体和属性读取 |
| REQ-METER-08 | 支持自定义单总功率、三相求和、总 import/export、三相 import/export 四类公式 |
| REQ-METER-09 | 电表实体应按电表类型放在普通 3EM / 自定义电表实体区，不应混在一体机高级实体覆盖中 |

## 9. 控制循环需求

蓝图应周期性执行闭环控制。

| 编号 | 需求 |
| --- | --- |
| REQ-CTRL-01 | 蓝图默认每秒触发一次 |
| REQ-CTRL-02 | 蓝图不应每秒无条件写入设备 |
| REQ-CTRL-03 | 蓝图应根据电表误差、负载口功率、PV、SOC 状态计算目标值 |
| REQ-CTRL-04 | 蓝图应同时考虑并网口功率设定和最大逆变输出设定 |
| REQ-CTRL-05 | 蓝图应等待控制写入后的新反馈，再继续下一次有效修正 |
| REQ-CTRL-06 | 蓝图应支持快、中、慢三档写入节奏 |
| REQ-CTRL-07 | 蓝图应支持单次最大调整步长限制 |
| REQ-CTRL-08 | 蓝图应支持写入分辨率，避免无意义小幅写入 |

## 10. SOC 需求

| 编号 | 需求 |
| --- | --- |
| REQ-SOC-01 | 蓝图应把用户设置的 SOC 下限写入设备自带下限实体 |
| REQ-SOC-02 | 蓝图应把用户设置的 SOC 上限写入设备自带上限实体 |
| REQ-SOC-03 | 蓝图应以用户设置的 SOC 上限作为满电保持进入条件 |
| REQ-SOC-04 | 蓝图应以用户设置的 SOC 下限作为低 SOC 保持进入条件 |
| REQ-SOC-05 | 蓝图内部应使用固定 `5%` 回差 |
| REQ-SOC-06 | 可选 Helper 存在时，蓝图应使用 Helper 保存满电或低 SOC 保持状态 |
| REQ-SOC-07 | 未配置 Helper 时，蓝图应直接根据当前 SOC 判断状态 |

## 11. 普通零馈电工况需求

普通工况指 SOC 未满、未到下限，且所有关键数据有效。

| 编号 | 需求 |
| --- | --- |
| REQ-MODE-01 | 电表显示馈电时，蓝图应降低输出或尝试吸收多余功率 |
| REQ-MODE-02 | 电表显示购电时，蓝图应提高输出以补足负载 |
| REQ-MODE-03 | PV 输入变化时，蓝图仍应以外部电表目标值为主 |
| REQ-MODE-04 | 负载口负载增加时，蓝图应提高最大逆变输出设定 |
| REQ-MODE-05 | 负载口负载降低时，蓝图应降低输出避免外送 |
| REQ-MODE-06 | 负载口回灌时，蓝图应纳入零馈电计算 |

## 12. 满电工况需求

满电后必须支持两种策略。

### 12.1 Follow load after full

| 编号 | 需求 |
| --- | --- |
| REQ-FULL-LOAD-01 | 满电后仍以外部电表接近目标值为优先 |
| REQ-FULL-LOAD-02 | 当 PV 大于负载时，允许限制 PV 或降低可用输出 |
| REQ-FULL-LOAD-03 | 负载增加时，应释放输出补负载 |
| REQ-FULL-LOAD-04 | 适用于严格零馈电优先场景 |

### 12.2 Follow PV after full

| 编号 | 需求 |
| --- | --- |
| REQ-FULL-PV-01 | 满电后应尽量让 PV 供应负载口 |
| REQ-FULL-PV-02 | 当 PV 大于负载时，剩余 PV 可能外送 |
| REQ-FULL-PV-03 | 蓝图应限制主动计算出的并网输出目标 |
| REQ-FULL-PV-04 | 文档必须说明满电 bypass 实际外送可能受硬件路径影响 |
| REQ-FULL-PV-05 | 若现场要求严格不外送，应建议使用 Follow load after full |

## 13. 低 SOC 工况需求

| 编号 | 需求 |
| --- | --- |
| REQ-LOW-01 | 低 SOC 保持时，蓝图应避免继续放电 |
| REQ-LOW-02 | 有 PV 时，最大逆变输出目标应尽量限制在可用 PV 范围内 |
| REQ-LOW-03 | 当负载大于 PV 时，允许外部电表显示购电 |
| REQ-LOW-04 | 蓝图不应为了零馈电突破用户设置的放电下限 |

## 14. 负载口工况需求

负载口为正：

| 编号 | 需求 |
| --- | --- |
| REQ-LP-LOAD-01 | 蓝图应把负载口消耗纳入总输出计算 |
| REQ-LP-LOAD-02 | 负载口消耗增加时，应提高最大逆变输出设定 |
| REQ-LP-LOAD-03 | 满电 Follow PV 时，PV 应优先供应负载口 |

负载口为负：

| 编号 | 需求 |
| --- | --- |
| REQ-LP-BACKFEED-01 | 蓝图应识别负载口回灌 |
| REQ-LP-BACKFEED-02 | 电池未满时，应尝试吸收或降低其他输出 |
| REQ-LP-BACKFEED-03 | 蓝图应在并网侧吸收目标上应用 AC 耦合最大充电功率限制 |
| REQ-LP-BACKFEED-04 | 文档必须说明负载口回灌路径不一定等同于并网口回充限制 |

## 15. 功率限制需求

| 编号 | 需求 |
| --- | --- |
| REQ-LIMIT-01 | 蓝图应限制主动计算的最大并网输出目标 |
| REQ-LIMIT-02 | 蓝图应支持并网口功率设定下限，用于限制允许的负向吸收能力 |
| REQ-LIMIT-03 | 蓝图应支持 AC 耦合最大充电功率 |
| REQ-LIMIT-04 | 蓝图应钳制写入值到实体允许范围内 |
| REQ-LIMIT-05 | 文档必须说明硬件实际功率可能与主动设置值存在差异 |

## 16. 数据有效性和保护需求

| 编号 | 需求 |
| --- | --- |
| REQ-SAFE-01 | 外部电表实体无效时，本轮不得写入 |
| REQ-SAFE-02 | 一体机关键实体无效时，本轮不得写入 |
| REQ-SAFE-03 | 电表、负载口、并网口、PV 或 SOC 数据超时时，本轮不得写入 |
| REQ-SAFE-04 | SOC 上下限实体不可用时，本轮不得写入 |
| REQ-SAFE-05 | 停止本轮写入不应关闭自动化 |
| REQ-SAFE-06 | 下一轮数据恢复后，蓝图应继续计算 |
| REQ-SAFE-07 | 三相求和模式下，在三相实体均存在且 state 有效时，只要任一相在超时时间内更新，应允许继续使用当前三相 state 求和 |

## 17. 高级设置需求

| 编号 | 需求 |
| --- | --- |
| REQ-ADV-01 | 高级实体覆盖默认折叠 |
| REQ-ADV-02 | 高级电表符号方向和单位倍率设置默认折叠 |
| REQ-ADV-03 | 高级控制设置默认折叠 |
| REQ-ADV-04 | 一体机高级实体覆盖留空时应继续自动绑定 |
| REQ-ADV-05 | 用户只需填写需要纠偏的一体机覆盖项 |

## 18. 可观测性要求

为支持用户排查和测试验证，蓝图相关文档应建议观察以下数据：

| 数据 | 用途 |
| --- | --- |
| 外部电表实时功率 | 判断是否接近目标 |
| 并网口功率设定 | 判断蓝图主动输出目标 |
| 最大逆变输出设定 | 判断总输出上限变化 |
| 负载口功率 | 判断负载或回灌 |
| 并网口实际功率 | 判断设备反馈是否跟随 |
| PV 功率 | 判断 PV 是否被利用或限制 |
| SOC | 判断满电或低 SOC 状态 |
| HA 自动化 trace | 辅助判断每轮条件、变量和写入动作 |
| HA 日志 | 辅助判断插件写入超时或实体异常 |

## 19. 验收标准

### 19.1 配置验收

| 编号 | 标准 |
| --- | --- |
| ACC-CONFIG-01 | 普通用户只配置必要项即可创建自动化 |
| ACC-CONFIG-02 | 常见一体机实体可自动绑定 |
| ACC-CONFIG-03 | 常见电表总功率可自动绑定 |
| ACC-CONFIG-04 | 自动绑定失败时可通过高级覆盖完成配置 |

### 19.2 控制验收

| 编号 | 标准 |
| --- | --- |
| ACC-CTRL-01 | 普通工况下，外部电表应收敛到目标值附近 |
| ACC-CTRL-02 | 外部负载突增后，蓝图应提高输出并重新收敛 |
| ACC-CTRL-03 | 外部馈电突增后，蓝图应降低输出或吸收并重新收敛 |
| ACC-CTRL-04 | 负载口回灌时，蓝图应识别并纳入计算 |
| ACC-CTRL-05 | 满电 Follow load 应优先减少外送 |
| ACC-CTRL-06 | 满电 Follow PV 应优先利用 PV，并说明剩余 PV 可能外送 |
| ACC-CTRL-07 | 低 SOC 时不应继续强制放电追零 |

### 19.3 安全验收

| 编号 | 标准 |
| --- | --- |
| ACC-SAFE-01 | 任一关键实体无效时，本轮不写入 |
| ACC-SAFE-02 | 数据超时时，本轮不写入 |
| ACC-SAFE-03 | 写入值不得超出实体范围 |
| ACC-SAFE-04 | 自动化重载后不应要求用户必须配置 Helper |

## 20. 文档要求

对外说明文档必须包含：

- 普通用户配置步骤。
- 电表方向说明。
- SOC 上下限说明。
- 满电两种策略的差异。
- 最大并网输出功率建议。
- 满电 Follow PV 可能外送的边界说明。
- 常见异常排查。

对外说明文档不得包含：

- 现场设备序列号。
- Home Assistant 账号密码。
- 私有网络地址。
- 测试用外部设备的真实标识。

## 21. 建议测试方案

本节不是产品需求，只是为了验证上述需求是否被蓝图正确实现。测试平台、人工测试或回归测试可以参考本节设计用例。

### 21.1 建议测试矩阵

建议测试平台至少覆盖：

| 场景 | 输入变化 | 期望 |
| --- | --- | --- |
| 普通零馈电 | 电表从购电变为馈电 | 输出下降或吸收，电表回到目标附近 |
| 普通补负载 | 电表从馈电变为购电 | 输出提高，电表回到目标附近 |
| PV 输入变化 | PV 上升或下降 | 保持以电表目标为主 |
| 负载口负载 | 负载口功率为正 | 最大逆变输出随负载变化 |
| 负载口回灌 | 负载口功率为负 | 识别回灌并减少外送 |
| 满电 Follow load | SOC 达上限，PV 大于负载 | 优先零馈电，允许限制 PV |
| 满电 Follow PV | SOC 达上限，PV 大于负载 | 优先 PV 利用，允许剩余外送 |
| 低 SOC | SOC 达下限 | 不强制放电追零 |
| 三相求和 | 一相更新、其他相不变 | 允许继续按三相 state 求和 |
| 数据超时 | 电表或设备实体不刷新 | 本轮不写入 |
| 写入超时 | 插件写入失败 | 日志可见，下一轮继续尝试 |

### 21.2 已验证测试结论

以下结论来自当前版本的测试过程，只用于指导后续回归和 7x24 测试任务，不应写成用户需求：

| 验证项 | 观察结论 |
| --- | --- |
| 未满电、PV 存在、外部电表自动绑定 | 可作为普通零馈电回归场景 |
| 外部设备向同回路馈电 | 可作为外部馈电扰动回归场景 |
| 外部设备作为负载 | 可作为负载突增回归场景 |
| 显式三相求和，只有一相实时更新 | 可作为三相数据新鲜度回归场景 |
| 满电 Follow load | 可作为严格零馈电优先回归场景 |
| 满电 Follow PV | 可作为 PV 优先利用和 bypass 边界回归场景 |
| 小并网输出限制 | 可作为主动设置值限制和硬件实际功率差异的观察场景 |

### 21.3 建议重点观察边界

| 边界 | 建议观察方式 |
| --- | --- |
| 满电 bypass | 观察实际外送是否与主动设定值存在差异 |
| 负载口回灌 | 观察负载口回灌是否完全受 AC 侧限制约束 |
| 低 SOC | 在安全低电量环境中补测，不建议强行消耗现场电池 |
| 插件写入超时 | 通过 HA 日志和自动化 trace 记录重试情况 |
| 电表实体命名 | 用不同电表插件验证自动绑定和高级覆盖 |

### 21.4 7x24 测试平台建议

测试平台接入该蓝图时，应满足：

- 不把外部模拟负载或外部发电设备配置进蓝图。
- 蓝图只绑定被测一体机和外部电表。
- 外部负载、外部发电、PV 调整应作为测试扰动源独立控制。
- 每个测试任务应记录自动化输入、关键实体状态、HA trace、HA 日志和最终恢复动作。
- 每轮测试结束后必须关闭测试自动化，并恢复一体机和外部扰动源到安全默认状态。
- 测试报告不得包含现场设备序列号、设备 ID、账号或私有地址。
